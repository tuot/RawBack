Description:
RawBack helps you get from raw source files back to the repository page they came from.

When you open a raw file from a supported Git host, RawBack adds a small, low-distraction navigation bar at the top of the page. One tap opens the same file in the normal repository view, so you can see the surrounding folder, branch, history, and project context.

Built for developers and code readers:
- Jump from raw files to repository file pages
- Works with GitHub, GitHub Gist, GitLab, Gitea, and Codeberg raw URLs
- Add self-hosted GitLab and Gitea domains
- Choose light, dark, or system appearance
- Enable only the Git hosts you use
- Hide individual files when you do not want the banner there
- Use RawBack in English, Chinese, Japanese, Korean, Spanish, French, German, and Russian

RawBack stays focused on one job: restoring context when a raw file leaves you outside the repository interface. It does not require an account, does not include ads, and does not send your browsing data to a server.

## App Review Information

Sign-In Required:
No

Demo Account:
Not applicable. RawBack does not require an account.

Review Notes:
RawBack is a Safari web extension for navigating from raw Git file URLs back to the corresponding repository file page. It does not require login, subscription, in-app purchases, or a demo account.

Testing steps:
1. Install the app and enable the RawBack extension in Safari.
2. Open a supported raw Git file URL, for example:
   - https://raw.githubusercontent.com/vuejs/core/main/README.md
   - https://gist.githubusercontent.com/octocat/1182433/raw/c26b772fb22934ef42bdad13e648c697b37f40eb/git-flow-commands.txt
   - https://gitlab.com/gitlab-org/gitlab/-/raw/master/README.md
   - https://gitea.com/gitea/tea/raw/branch/main/README.md
3. Confirm that the RawBack banner appears at the top of the raw file page.
4. Tap or click "View in repository" and confirm that Safari opens the matching repository file page.
5. Open the RawBack extension popup to verify the enabled switch, theme setting, language setting, platform toggles, hidden files, and custom domain settings.

The extension requests broad website access so it can detect raw Git file URLs across supported public and self-hosted Git services. RawBack uses this access locally in the browser to detect matching pages and inject the navigation banner. It does not send browsing data to a server.
